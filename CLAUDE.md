# CLAUDE.md

## Project Overview

ralph-node is a template repository implementing the Ralph Loop — an automated, test-gated AI development cycle for Node.js projects. An LLM agent reads tasks from a PRD, implements code, and an external orchestrator validates the work against tests before committing. The loop repeats until all tasks pass.

This repo contains only the loop infrastructure. Application code is generated by AI agents executing PRDs.

## Bootstrap

The project must be initialized before first use:

```bash
nvm use           # Node v24.14.1 (see .nvmrc)
bash init.sh      # Creates package.json, installs deps, moves init PRD to PRD.md, runs the Ralph Loop, self-destructs
```

After init, `npm run ralph` starts the loop for subsequent PRDs.

## Commands

- `npm test` — Run Jest unit tests then Playwright E2E tests (sequential)
- `npm run ralph` — Start the Ralph Loop (reads PRD.md, implements tasks, validates, commits)
- `npm run backpressure` — Generate test files from PRD.md (run before ralph to create test backpressure)
- `npm run lint` — Lint & Format with Biome (auto-fix)

## Code Style

- Formatter/linter: Biome (see `biome.json`)
- 2-space indentation, 100-char line width
- Double quotes, trailing commas, always semicolons
- `app/` and `pages/` directories: default exports are allowed (Next.js convention)

## Testing

- **Unit tests:** Jest — `.test.ts` files for logic-only tasks
- **E2E tests:** Playwright — `.spec.ts` files for UI tasks
- Use `data-testid` attributes as element selectors. Do not use class names or CSS selectors.
- Every test must fail against a blank/missing implementation. No trivially passing assertions.
- Minimum coverage per test file: happy path + most likely failure/edge case

## Ralph Loop Architecture

### How it works

1. **PRD.md** contains checkbox tasks (e.g., `- [ ] Create landing page`)
2. **Backpressure** (`npm run backpressure`) generates failing tests for each task
3. **Ralph** (`npm run ralph`) iterates: agent reads PRD + context, implements code, orchestrator runs tests
4. On pass: task is checked off, files committed, memory/ledger updated
5. On fail: error output is fed back to the agent for retry
6. When all tasks pass: PRD is archived to `.prds/PRD.N.md`

### PRD task format

```markdown
- [ ] Task description `[test: npx jest tests/foo.test.ts]`
```

The `[test: command]` annotation specifies targeted validation for that task. If omitted, `npm test` (full suite) is used.

### Key files (do not modify during ralph execution)

| File | Purpose |
|------|---------|
| `.github/scripts/ralph.sh` | Main loop orchestrator |
| `.github/scripts/backpressure.sh` | Test generation script |
| `.github/prompts/ralph.md` | System prompt injected into the agent each cycle |
| `.agent-ledger.jsonl` | Append-only log of completed tasks (JSON lines) |
| `MEMORY.md` | Agent scratchpad — overwritten each cycle, deleted on PRD completion |
| `PRD.md` | Active task list — gitignored, archived when complete |

### Files the agent must never modify

- Anything in `.github/scripts/` or `.github/prompts/`
- Test assertions or mock logic (to force a pass)
- `.claude/settings.json`, `.aignore`, `biome.json`

## Writing PRDs

PRDs drive all development. When writing a new PRD:

- Each task should be a single, atomic unit of work
- Include a `[test: command]` annotation when possible to enable targeted backpressure
- If a task requires new dependencies, state them explicitly (e.g., "Install next, react, react-dom") — agents cannot add dependencies unless the PRD instructs it
- Order tasks so earlier tasks establish structure that later tasks build on
- Keep tasks small enough that a single agent cycle can complete them

## Conventions

- Commit messages from the loop follow the format: `chore(ai): <task description>`
- Shell scripts use `set -euo pipefail` with explicit `set +e` around commands expected to fail (test validation, engine invocation)
- The `.aignore` file prevents AI tools from indexing ralph infrastructure — this is intentional separation between the loop and the code it produces
